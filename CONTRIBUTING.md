# Contributing to the Cromer Narrative Engine

## The Pantomime Protocol (v1.1)
This repository contains the reference implementation of the **Pantomime Protocol**. All contributions must align with the project's focus on high-density, state-managed narrative generation.

## Technical Standards
To maintain the integrity of the narrative engine, submissions must satisfy these requirements:
* **Mathematical Consistency**: Changes to the core logic must account for **Causal Debt ($D_c$)** and the **Composition Operator ($\oplus$)**.
* **Type Safety**: All updates to NPC logic or environmental interactions must be reflected in `src/types.ts` using the **Tiered Physics** payload structure.
* **Telemetry**: Any new features should be accompanied by updated JSON files in the `telemetry/` directory to document execution during testing.

## How to Submit
1. Fork the repository.
2. Create a feature branch focusing on a specific element of the **Narrative Credit System** or **Semantic Facade**.
3. Ensure your code does not break the "Causal Roast" telemetry case study.
4. Submit a Pull Request with a detailed description of how the change impacts narrative state management.
